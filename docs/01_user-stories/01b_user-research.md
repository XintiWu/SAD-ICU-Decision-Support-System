## 2. 使用者角色（User Personas）

### 2.1 護理師（Nurse）

**典型情境**
> 每班接手前，需要了解自己負責哪些病患、有哪些待辦任務，並協助填寫病患麻煩度評估。班中需追蹤任務完成狀況，並即時回應突發 STAT 醫囑。

| 項目 | 說明 |
|------|------|
| 主要目標 | 清楚知道自己負責的病患與工作量，有效率地完成班中任務 |
| 使用頻率 | 每班（約每 8 小時） |
| 主要操作 | 填寫麻煩度量表、查看待辦清單、核銷完成任務 |
| 痛點 | 填表流程繁瑣；不清楚自己的工作負荷相對其他人是否合理 |

**核心需求**
- 快速填寫主觀麻煩度，不重複輸入客觀資料
- 清楚看到自己這班的病患與剩餘工作量
- 及時收到 STAT 突發醫囑通知

---

### 2.2 小組長（Charge Nurse）

**典型情境**
> 每班交班前，需要根據護理師出勤狀況與病患麻煩度，在短時間內完成分床。過去靠經驗與紙本試算，容易遺漏或出現負荷不均的狀況。班中還需隨時掌握各護理師工作進度，處理突發狀況。

| 項目 | 說明 |
|------|------|
| 主要目標 | 快速完成公平的分床，班中維持全局視野、即時調度人力 |
| 使用頻率 | 每班（通常為資深護理師輪值） |
| 主要操作 | 確認出勤人員、產生建議分床、拖曳調整、確認分床、查看戰情室、追蹤 STAT |
| 痛點 | 分床決策需考量因素多（負荷平衡、床位鄰近、護理師年資）；缺乏即時全局視角 |

**核心需求**
- 系統自動整合麻煩度分數、床位位置、護理師年資，給出可參考的建議
- 能在建議基礎上快速調整，不需從頭計算
- 戰情室即時呈現各護理師負荷，無需逐一詢問

---

### 2.3 醫師（Doctor）

**典型情境**
> 在查房或臨床決策後，需要快速建立 STAT 突發醫囑，確保護理師即時接收並處理。

| 項目 | 說明 |
|------|------|
| 主要目標 | 快速輸入突發醫囑，確認被指定護理師看到並追蹤完成狀態 |
| 使用頻率 | 不定期（依臨床狀況） |
| 主要操作 | 新增 STAT 醫囑 |
| 痛點 | 現有溝通依賴口頭或紙本，容易遺漏或延誤 |

**核心需求**
- 快速建立 STAT 醫囑並連結至特定病患
- 確認醫囑已進入護理師的待辦流程

## 3. 使用者痛點分析（Before / After）

### 3.1 班前分床流程

| 痛點（Before） | 系統解法（After） |
|--------------|----------------|
| 小組長靠經驗與記憶試算負荷，耗時且易出錯 | 系統自動計算麻煩度分數並產生建議分床，秒內完成 |
| 護理師填表資訊不互通，小組長需逐一確認 | 所有護理師填寫資料即時彙整，小組長統一查看 |
| 分床僅靠白板或口頭宣布，易遺漏 | 確認後系統鎖定並公開顯示，所有人看到同一版本 |
| 未考慮床位鄰近性，護理師班中走動距離大 | 演算法以走廊距離為約束，優先分配鄰近床位 |

### 3.2 班中追蹤流程

| 痛點（Before） | 系統解法（After） |
|--------------|----------------|
| 小組長不清楚各護理師當下的剩餘工作量 | 戰情室即時顯示每位護理師的未完成任務與負荷分數 |
| STAT 醫囑依賴口頭傳達，容易遺漏 | STAT 醫囑直接推入護理師待辦清單，戰情室同步顯示 |
| 無法快速評估是否需要調度人力 | 戰情室以顏色視覺化負荷高低，協助決策 |

### 3.3 交班流程

| 痛點（Before） | 系統解法（After） |
|--------------|----------------|
| 交班資料依賴手寫或口頭，易遺漏資訊 | 系統在確認分床時自動封存交班快照 |
| 歷史班別分配無從查詢 | 交班快照保存於資料庫，可隨時查詢歷史記錄 |

## 4. 使用者旅程（User Journey）

### 4.1 小組長班前分床旅程

```
[班前 30 分鐘]
    │
    ▼
確認當班護理師
（RosterImportPage / 班表匯入）
    │
    ▼
護理師填寫麻煩度評估
（BurdenFormPage）
    │
    ▼
小組長查看評估進度 → 提醒未填寫護理師
    │
    ▼
點擊「產生建議分床」
（ChargeAllocationPage）
    │
    ├── 查看決策紀錄了解演算法邏輯
    │
    ▼
視需要拖曳調整分配
（即時更新各護理師負荷顯示）
    │
    ▼
確認分床 → 系統鎖定 + 產生交班快照
    │
    ▼
[班次開始]
```

### 4.2 護理師班中任務旅程

```
[班次開始]
    │
    ▼
查看本班分配病患
（NurseOverviewPage）
    │
    ▼
查看待辦任務 + STAT 醫囑
（NurseTodoPage）
    │
    ├── 新 STAT 醫囑隨時推入
    │
    ▼
逐一核銷完成任務
（更新任務狀態）
    │
    ▼
[交班前]
    │
    ▼
確認任務全數完成或標記延後
```
