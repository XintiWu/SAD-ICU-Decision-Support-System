# User Stories Mapping

## 使用者角色

| 角色 | 說明 |
|------|------|
| **護理師 (Nurse)** | 負責填寫病患麻煩度評估、查看個人待辦任務與分床結果 |
| **小組長 (Charge Nurse)** | 負責產生分床建議、調整並確認分床、查看戰情室與交班快照 |
| **醫師 (Doctor)** | 新增突發 STAT 醫囑 |

---

## Epic 一覽

| # | Epic | 說明 |
|---|------|------|
| E1 | 班別與班表管理 | 匯入護理師班表、確認當班人員 |
| E2 | 麻煩度評估 | 填寫與送出病患負荷量表 |
| E3 | 分床決策 | 產生建議分床、人工調整、確認分床 |
| E4 | 護理師待辦清單 | 查看個人任務與 STAT 醫囑、核銷完成 |
| E5 | 小組長戰情室 | 即時監控各護理師負荷與工作進度 |
| E6 | STAT 突發醫囑 | 新增與追蹤突發醫囑 |
| E7 | 交班快照 | 封存分床確認當下的交班資料 |

## User Stories Mapping

### E1 — 班別與班表管理

| ID | 使用者故事 | 角色 | 優先度 | 實作狀態 |
|----|-----------|------|--------|---------|
| E1-1 | 身為小組長，我想要匯入 xlsx 護理師班表，以便快速建立當班護理師名單，不需手動輸入。 | 小組長 | 高 | ✅ 完成 |
| E1-2 | 身為小組長，我想要查看當班出勤護理師清單，以便確認人力配置後再進行分床。 | 小組長 | 高 | ✅ 完成 |
| E1-3 | 身為系統，我想要記錄每班的起訖時間與班別類型（白班／小夜／大夜），以便後續分床與交班查詢有明確的班別依據。 | — | 高 | ✅ 完成 |

### E2 — 麻煩度評估

| ID | 使用者故事 | 角色 | 優先度 | 實作狀態 |
|----|-----------|------|--------|---------|
| E2-1 | 身為護理師，我想要填寫每位病患的客觀麻煩度因素（如 RASS 分數、用藥複雜度），以便系統自動計算客觀負荷分數。 | 護理師 | 高 | ✅ 完成 |
| E2-2 | 身為護理師，我想要填寫每位病患的主觀麻煩度判斷，以便反映臨床現場實際感受到的照護難度。 | 護理師 | 高 | ✅ 完成 |
| E2-3 | 身為護理師，我想要先儲存草稿（draft）再正式送出（submitted），以便分次填寫不遺失進度。 | 護理師 | 中 | ✅ 完成 |
| E2-4 | 身為小組長，我想要在分床前看到哪些病患的麻煩度尚未填寫完整，以便提醒護理師補填，確保分床建議品質。 | 小組長 | 中 | ✅ 完成 |
| E2-5 | 身為系統，我想要即時加總每位病患的麻煩度總分（objective + subjective），以便分床演算法使用最新數據。 | — | 高 | ✅ 完成 |

### E3 — 分床決策

| ID | 使用者故事 | 角色 | 優先度 | 實作狀態 |
|----|-----------|------|--------|---------|
| E3-1 | 身為小組長，我想要一鍵產生系統建議分床，以便在數秒內得到依負荷平衡與床位鄰近性計算出的分配方案，不需手動試算。 | 小組長 | 高 | ✅ 完成 |
| E3-2 | 身為小組長，我想要透過拖曳介面手動調整分配，以便在系統建議的基礎上加入臨床判斷。 | 小組長 | 高 | ✅ 完成 |
| E3-3 | 身為小組長，我想要在調整時即時看到各護理師的負荷變化，以便評估調整是否合理。 | 小組長 | 高 | ✅ 完成 |
| E3-4 | 身為小組長，我想要查看系統的分床決策紀錄（每位病患的候選護理師與最終選擇原因），以便了解建議邏輯並建立信任。 | 小組長 | 中 | ✅ 完成 |
| E3-5 | 身為小組長，我想要確認分床後系統鎖定結果，以便所有護理師看到一致的最終分配。 | 小組長 | 高 | ✅ 完成 |
| E3-6 | 身為小組長，我想要在確認分床前看到「仍有未分配病患」的警告，以避免遺漏病患。 | 小組長 | 高 | ✅ 完成 |
| E3-7 | 身為小組長，我想要在需要時退回分床草稿重新調整，以便因應突發人力或病患異動。 | 小組長 | 中 | ✅ 完成 |
| E3-8 | 身為護理師，我想要查看本班最終分床結果，以便知道自己負責哪些病患。 | 護理師 | 高 | ✅ 完成 |
